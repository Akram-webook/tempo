# Tempo — Fast, no-waiting workflow

Goal: change something and immediately move on — never sit waiting on a build or a deploy.

## One-time setup
```bash
npm install            # installs jsdom (only dev dependency, used by the tests)
```
Then enable CI deploys once: GitHub repo → **Settings → Pages → Build and deployment →
Source = GitHub Actions**. (The workflow is already in `.github/workflows/deploy.yml`.)

## The two-terminal loop
- **Terminal 1 — auto-build (leave running):**
  ```bash
  npm run watch
  ```
  Rebuilds `dist/index.html` the instant you save anything under `src/`. You never run build by hand.
- **Terminal 2 — your commands:**
  ```bash
  npm test           # run the jsdom checks when you want to verify
  git add -A && git commit -m "..." && git push   # push = auto-deploy via CI
  ```

That's it. Edit → save (auto-builds) → push (auto-deploys). Nothing blocks you, and you can
start the next change immediately.

## Don't-wait habits
- **Branch per task:** `git switch -c feature/x` → work → push → open PR. Independent tasks proceed
  in parallel and never block each other.
- **Let CI deploy:** after `git push`, don't watch it — GitHub Actions builds, tests, and ships in
  the background. Check the green check later.
- **Local preview:** use the VS Code "Live Server" extension (no terminal) or `npx serve .`.

## Rules
- Edit `src/**` only. `dist/index.html` is generated by `build.js` — never hand-edit it.
- `npm test` must pass before you push to `main` (CI also runs it and will block a red build).
- Never commit secrets. The Supabase URL + `sb_publishable_…` key are public-safe; the
  `service_role` key must never touch the front-end or the repo.
