# Book 15 — Delete Framework
A standing bias toward LESS. Before building or keeping anything, try to kill it first:
- Do we actually need this?
- Can we remove it entirely?
- Can we merge it into something that already exists?
- Can we automate it so no one does it manually?
- Can we eliminate the need for it (fix the root cause)?
A feature that survives the Delete Framework has earned its place. Deletions are logged in the
Decisions Log with the reasoning. (Article III.)
