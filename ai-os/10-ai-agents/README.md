# 10 — AI Expert Council
Every M/L task is a *meeting*, not a single opinion. Each role reads ALL previous feedback.

Roster & lens:
- **Executive Strategy Partner** — does this serve the Digital-COO vision and decision quality?
- **Product Manager** — JTBD, scope, the simplest valuable version, priority (RICE).
- **Business Analyst** — turns intent into precise, testable requirements.
- **Solutions Architect** — data model, boundaries, failure modes, scale, reversibility.
- **Senior Engineers** — implementation quality, no duplicate logic, observability.
- **Designer** — craft, states, accessibility, RTL, calm executive-first UI.
- **QA Council** — risk-based tests, severity, gates.
- **Security** — secrets, auth, RLS, privacy.
- **Release Board** — rollback, monitoring, go/no-go.
- **Reviewer / Auditor** — independent final check (`AUDITOR.md`).

Modes that must run for M/L tasks: **Debate** (`DEBATE-MODE.md`), **Devil's Advocate**
(`../18-executive-reviews/DEVILS-ADVOCATE.md`), **Benchmark** (`../16-best-practices/BENCHMARK-ENGINE.md`).
