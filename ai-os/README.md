# Tempo AI Operating System (Tempo-OS)
The constitution of Tempo. Every contributor — human or AI — reads this before work.

Tempo is not a dashboard. It is the **operating brain of Event Operations**, on the path to a
**Digital Chief Operating Officer**: it observes operational signals, analyzes workload and
execution, *challenges* decisions, recommends improvements, predicts risk, and helps leaders decide
*before* problems happen. See `VISION.md`.

## How to use this (start here)
1. Every task enters through **`19-ai-orchestrator/`** — it routes the task and *right-sizes* the rigor.
2. The orchestrator pulls the relevant **standards, templates, checklists, and scorecards** below.
3. Output ends with a **Product Health Score** (`14-scorecards/`) and, for features, an
   **Executive Report** (`18-executive-reviews/`).

## Right-sizing rule (anti-bureaucracy — non-negotiable)
Match process to task size so the OS accelerates quality instead of slowing it:
- **XS (typo, copy, css tweak):** Engineering + QA checklist only.
- **S (small fix/UI change):** + Design review + scorecard (light).
- **M (feature):** full Expert Council + Debate + Devil's Advocate + Benchmark + scorecard + Executive Report.
- **L (architecture/data/security change):** all of M + Architecture review + Risk Engine + Auditor.
