# Book 18 — Anti-Pattern Library (things Tempo will NEVER do)
- Never measure people by hours.
- Never use utilization as a performance score.
- Never build dashboards nobody opens.
- Never collect data without a decision attached to it (Article IV).
- Never ship a recommendation that can't be explained/traced (Article V).
- Never reward unhealthy behavior or enable surveillance (Article II).
- Never add complexity without a decision-quality gain (Article III).
- Never hand-edit generated output (`dist/`); never commit secrets.
- Never let an AI agent merge to `main` without Akram's OK.
Add new entries whenever a review catches a pattern worth banning forever.
